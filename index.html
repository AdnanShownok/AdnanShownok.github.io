01 About

My research sits at the intersection of machine learning and applied cybersecurity. I work on systems that don't just classify network traffic as malicious or benign, but explain why, ground that explanation in established security frameworks, and propose mitigation — closing the loop from detection to actionable response.

My undergraduate thesis, supervised by Prof. Dr. Kazi A. Kalpoma, built a two-stage framework: classical ML classifiers (XGBoost, Random Forest, LightGBM) for fast, accurate detection, paired with a confidence-aware retrieval-augmented pipeline and a four-agent LLM architecture for incident analysis grounded in MITRE ATT&CK and NIST SP 800-53.

Outside the thesis, I've built full-stack and desktop applications in PHP and Java, and I'm continuing to work across the ML, security, and open-source software stack.

03 Projects
Hybrid ML + LLM Intrusion Detection System
June 2025 – June 2026
Python XGBoost LLaMA RAG Pinecone
  • Designed a two-stage cybersecurity framework combining ML classifiers (XGBoost, Random Forest, LightGBM) with a multi-agent LLM pipeline for attack detection, explanation, and mitigation across three benchmark IDS datasets.
  • Engineered a confidence-aware adaptive RAG pipeline that routes IDS incidents by prediction confidence, activating Top-3 multi-class retrieval for ambiguous low-confidence cases — 97.6% of all misclassifications occurred in this low-confidence region, validating the routing design.
  • Built a 4-agent LLM architecture (Orchestrator, Worker, Mitigation, Synthesizer) producing structured 8-section threat assessment reports grounded in MITRE ATT&CK and NIST SP 800-53 knowledge bases.
  • Evaluated report quality with an LLM-as-judge methodology across 540 test incidents; LLaMA-3.1-8B scored consistently 4.64–4.76 / 5 across all datasets.
